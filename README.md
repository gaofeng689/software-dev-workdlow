# Claude Skills Collection

一组专为 Claude AI 助手设计的技能集合，覆盖软件开发全流程、Bug 修复和功能测试等核心场景。

## 📦 技能列表

### 1. Software Development Workflow（软件开发工作流）

**位置**: `software-dev-workflow/SKILL.md`

完整的软件开发流程指南，遵循 **"规划 → 执行 → 验证 → 迭代"** 的开发范式。

**核心原则**:
- 先思后行：充分理解需求再动手
- 小步快跑：大任务拆分为小任务
- 持续验证：每个节点都要验证
- 闭环交付：确保任务通过测试

**适用场景**:
- 新功能开发
- 需求分析与任务拆分
- 代码重构和优化
- 系统架构设计

**触发方式**: 用户提出开发需求、要求实现功能、进行代码重构等

---

### 2. Bug Fix（Bug 修复）

**位置**: `bug-fix/SKILL.md`

系统化的调试和修复流程，确保 bug 被彻底修复而非表面掩盖。

**核心流程**:
1. **复现问题** (Reproduce) - 确认 bug 真实存在
2. **定位问题** (Locate) - 基于症状推断原因
3. **分析根因** (Analyze) - 找到根本原因
4. **修复问题** (Fix) - 精准改动，只改必需部分
5. **验证修复** (Verify) - 确保修复正确完整

**核心原则**: 不主观臆断，不掩饰困惑，清晰列明取舍

**适用场景**:
- 程序异常/报错
- 逻辑错误
- 功能不符合预期
- 数据计算错误

**触发方式**: "修复 bug"、"fix bug"、"报错了"、"程序崩溃"等

---

### 3. Test Assist（功能测试助手）

**位置**: `test-assist/SKILL.md`

自动对指定功能进行功能验证测试，发现问题后自动尝试修复。

**工作流程**:
```
功能理解 → 测试计划制定 → 功能验证执行 → 问题识别与修复 → 总结报告
```

**测试类型**:
- 代码审查
- 动态测试（API 调用、方法执行、前端模拟）
- 边界条件验证
- 异常处理测试

**输出格式**: 功能测试报告（包含测试结果、问题列表、修复验证）

**触发方式**: "测试某个功能"、"验证 XX"、"检查是否正常工作"等
