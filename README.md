# Claude Skills Collection

一组专为 Claude AI 助手设计的技能集合，覆盖软件开发全流程、Bug 修复、功能测试和技术学习等核心场景。

## 📦 技能列表

### 1. Software Development Workflow（软件开发工作流）

**位置**: `software-dev-workflow/SKILL.md`  
**版本**: 1.0.0

完整的软件开发流程指南，遵循 **"规划 → 执行 → 验证 → 迭代"** 的开发范式。

**核心原则**:
- 先思后行：充分理解需求再动手
- 小步快跑：大任务拆分为小任务
- 持续验证：每个节点都要验证
- 闭环交付：确保任务通过测试

**六阶段流程**:
1. **头脑风暴** - 澄清需求、分析可行性
2. **任务定义与拆分** - 定义 Scope、拆分子任务、定义完成标准
3. **Subagent 决策** - 分析依赖、分配任务
4. **开发与迭代** - 编码规范、持续重构、性能优化
5. **编译测试** - 单元测试、集成测试、边界测试
6. **迭代优化** - 问题分析、修复策略、回归测试

**触发方式**: "开发一个用户认证模块"、"实现文件上传功能"、"帮我写个 XX"

### 2. Bug Fix（Bug 修复）

**位置**: `bug-fix/SKILL.md`  
**版本**: 1.0.0

系统化的调试和修复流程，确保 bug 被彻底修复而非表面掩盖。

**核心原则**: 不主观臆断，不掩饰困惑，清晰列明取舍

**五步修复流程**:
1. **复现问题** (Reproduce) - 收集错误信息、最小化复现用例
2. **定位问题** (Locate) - 分析错误类型、使用工具定位、缩小范围
3. **分析根因** (Analyze) - 问为什么、识别偶发问题
4. **修复问题** (Fix) - 精准改动、只改必需部分
5. **验证修复** (Verify) - 重新测试、边界验证、添加测试用例

**适用场景**:
- 程序异常/报错
- 逻辑错误
- 功能不符合预期
- 数据计算错误

**触发方式**: "修复 bug"、"报错了"、"程序崩溃"、"不对"、"fix this"

### 3. Test Assist（功能测试助手）

**位置**: `test-assist/SKILL.md`  
**版本**: 1.0.0

自动对指定功能进行功能验证测试，发现问题后自动尝试修复。

**工作流程**:
```
功能理解 → 测试计划制定 → 功能验证执行 → 问题识别与修复 → 总结报告
```

**测试类型**:
- 代码审查
- 动态测试（API 调用、方法执行、前端模拟）
- 边界条件验证
- 异常处理测试

**输出格式**: 功能测试报告（测试结果、问题列表、修复验证）

**触发方式**: "测试一下这个功能"、"验证 XX 是否工作"、"帮我测试"

### 4. Tech Learn（技术学习助手） ⭐ 新增

**位置**: `tech-learn/SKILL.md`  
**版本**: 1.0.0

技术学习助手，主动查询官网、GitHub、技术文档，提供详细学习路径和原理解释。

**核心原则**:
- 权威优先：官网、官方文档、GitHub 一手资料
- 实践导向：可操作的实践步骤
- 防坑指南：指出新手常见错误
- 循序渐进：从入门到深入的学习路径

**执行流程**:
1. **识别技术名称和需求** - 提取技术名、具体需求、用户背景
2. **制定学习计划** - 入门学习/原理探究/实践应用/对比选型
3. **收集资料** - 官方文档 → GitHub → 权威技术文档 → 可信博客
4. **输出结构** - 概述 → 核心概念 → 学习路径 → 实践步骤 → 避坑指南

**输出内容**:
- 📌 概述：一句话介绍是什么、解决什么问题
- 🏗️ 核心概念：3-5 个最重要的概念
- 📚 学习路径：入门 → 进阶 → 精通
- 💻 实践步骤：详细的操作指南
- ⚠️ 常见坑和避坑指南：3-5 个最常见的坑
- 📖 参考资源：官方文档、GitHub、优质资源

**触发方式**: "了解 XX 技术"、"学习 XX"、"XX 是怎么实现的"、"什么是 XX"
