# Software Development Workflow Skill

一个完整的软件开发全流程工作流指南，遵循 **"规划 → 执行 → 验证 → 迭代"** 的开发范式。

## 📋 简介

这个 Skill 为 AI 助手提供了一套系统化的软件开发方法论，确保从需求分析到最终交付的每个环节都经过充分思考和验证。适用于各种软件开发场景，包括需求分析、项目构建、功能开发、代码重构和测试验证。

## 🎯 核心原则

| 原则 | 说明 |
|------|------|
| **先思后行** | 充分理解需求再动手，避免返工 |
| **小步快跑** | 大任务拆分为小任务，逐步完成 |
| **持续验证** | 每个节点都要验证，而非最后一次性测试 |
| **闭环交付** | 确保每个任务真正完成并通过测试 |

## 🔄 开发流程

### 第一阶段：头脑风暴
充分理解需求，澄清疑问，明确边界
- 澄清需求：确认输入/输出、使用场景、约束条件
- 分析可行性：技术可行性、参考方案、依赖资源
- 输出：形成清晰的需求理解文档

### 第二阶段：任务定义与拆分
将大任务拆分为可管理的小任务
- 定义任务范围：明确 Scope In/Out，设定优先级
- 拆分任务项：3-7 个子任务，每个 15-60 分钟可完成
- 定义完成标准：具体、可验证的 Acceptance Criteria

### 第三阶段：Subagent 决策
决定是否启用 Subagent 来并行处理独立任务
- 分析任务依赖关系
- 根据任务独立性决定是否并行
- 合理分配主 Agent 和 Subagent 的职责

### 第四阶段：开发与迭代
按计划执行开发，持续调整优化
- 遵循编码规范，保持代码可读性
- 持续重构，识别并改进代码坏味道
- 性能优化，减少不必要的计算和 I/O
- 进度跟踪，及时调整计划

### 第五阶段：编译测试
验证所有任务是否符合预期
- 编译检查：语法、依赖、构建
- 单元测试：确保测试覆盖率
- 集成测试：验证模块间集成
- 边界测试：极端情况和错误处理

### 第六阶段：迭代优化
测试失败时持续修改直到通过
- 问题分析：区分代码错误、测试错误、需求错误
- 修复策略：针对性修复
- 回归测试：确保修复未引入新问题
- 循环迭代：直到所有测试通过

## ✅ 检查清单

每个阶段完成后都有对应的检查清单，确保：
- 需求清晰无歧义
- 任务拆分合理，完成标准明确
- 代码符合规范，进度符合预期
- 所有测试通过，可以交付

## 📄 输出格式

完成整个工作流后，会生成**开发完成报告**，包括：
- 任务概述
- 完成的任务项列表
- 测试结果（单元测试、集成测试、代码覆盖率）
- 遗留问题说明（如有）
- 后续建议（可选）

## 🚀 使用场景

- 开发新功能模块
- 代码重构和优化
- 系统架构设计
- 测试用例编写
- 技术难题攻关

## 📝 注意事项

1. **不要跳步**：严格按照流程执行
2. **及时沟通**：有问题及时与用户沟通
3. **保持灵活**：根据实际情况调整流程
4. **质量优先**：追求高质量交付
5. **文档优先**：重要决策和设计记录下来
