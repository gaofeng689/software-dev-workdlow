# Tech Learn Skill

当你想要了解、学习某个技术，或询问某个技术的实现原理时，使用这个skill。

## 核心原则

1. **权威优先**：先去官网、官方文档、GitHub官方仓库查找一手资料
2. **实践导向**：不仅讲理论，更要给出可操作的实践步骤
3. **防坑指南**：指出新手常犯的错误和容易踩的坑
4. **循序渐进**：从入门到深入，给出学习路径

## 执行流程

### 1. 识别技术名称和需求

从用户输入中提取：
- 技术名称（如 React、Redis、Kubernetes）
- 用户的具体需求（入门？原理？实践？对比？）
- 用户的背景（如果有的话）

### 2. 制定学习计划

根据需求决定重点：
- **入门学习**：安装配置 → 核心概念 → 简单示例 → 进阶特性
- **原理探究**：架构设计 → 核心算法 → 源码分析 → 设计模式
- **实践应用**：适用场景 → 配置优化 → 最佳实践 → 避坑指南
- **对比选型**：多个技术的对比 → 各自优缺点 → 适用场景

### 3. 收集资料

使用以下资源（按优先级）：
1. **官方文档**：官网、官方文档站点
2. **GitHub**：官方仓库、README、官方示例
3. **权威技术文档**：MDN、Spring官方文档等
4. **可信博客**：高质量技术博客（慎选，优先官方）
5. **视频教程**：官方或知名博主（辅助）

具体步骤：
- 使用 `WebSearch` 搜索官方文档和GitHub
- 使用 `WebFetch` 获取关键页面的详细内容
- 整理收集到的信息

### 4. 输出结构

按以下结构组织输出：

```markdown
# [技术名称] 学习指南

## 📌 概述
一句话介绍这是什么、用来解决什么问题、为什么需要它。

## 🏗️ 核心概念
列出最重要的3-5个核心概念，简要解释。

## 📚 学习路径

### 入门阶段
- 资源链接 + 具体要学什么
- 第一个Hello World的步骤

### 进阶阶段
- 核心特性的深入理解
- 配置项说明

### 精通阶段
- 高级特性、原理分析
- 性能优化、最佳实践

## 💻 实践步骤
按步骤来的详细操作指南（可以代码块形式）。

## ⚠️ 常见坑和避坑指南
列出3-5个最常见的坑：
1. **坑1**：描述 + 为什么容易踩 + 如何避免
2. **坑2**：...
3. **坑3**：...

## 📖 参考资源
- 官方文档：链接
- GitHub仓库：链接
- 其他优质资源：链接
```

### 5. 补充信息

如果用户没有指定具体问题，还可以主动提供：
- 该技术的最新版本动态
- 与同类技术的对比
- 就业/使用市场需求情况
- 学习资源推荐（书籍、视频等）

## 注意事项

1. **不说废话**：直接给出有价值的信息，不要车轱辘话
2. **信息准确**：从官方渠道获取信息，避免二手错误
3. **因人而异**：根据用户背景调整讲解深度
4. **持续更新**：提醒用户技术发展快，建议关注官方更新
